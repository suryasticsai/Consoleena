The page is
the filesystem.

Every other browser devtool, console library or inspector panel treats your app as a black box to poke at. Consoleena does one thing no other drop-in script does: it mounts your running page as a virtual Unix filesystem.

DOM, styles, globals, storage, network traffic, console logs — all become readable paths. That single decision means every Unix reflex you already have works on a live website. cat /network | grep 401 finds unauthorized requests instantly. find /dom -name input lists every input element. No DevTools clicking required.

From that same shell, you drive a Playwright-style locator engine, an LLM agent that plans and iterates, and a GitHub PR workflow — all client-side, no server needed.

Real Unix pipes over your DOM

Playwright locators, in the browser

Auto-waiting chainable locators with text=, role=, aria=. fill uses the native setter, so React controlled inputs update for real.

AI that operates the shell

/install

One tag. Works on any page, any framework, no build step required.

index.html
<script src="consoleena-v2.js"></script>
<!-- or from jsDelivr CDN -->
<script src="https://cdn.jsdelivr.net/gh/suryasticsai/Consoleena@main/consoleena-v2.js"></script>

Press Ctrl+Shift+K to open the panel. Both window.Consoleena and the short alias window.cona are registered.

/shell & filesystem

The page is mounted as a virtual read-through filesystem. Navigate it with commands you already know; pipe between them with |.

terminal
# browse the filesystem
ls /
tree -L 2 /js

# find things on the live page
cat /js/vars | grep -i token
find /dom -name button
cat /network | grep -v " 200 "   # non-200 requests

# mutate globals, call functions
set featureFlags.darkMode true
run calculateTotal 40 2
eval 'document.title = "patched"'

/recorder → codegen

Record real interactions, export a runnable Playwright test or a Consoleena replay script.

terminal
record start        # flip on the recorder — now click around the page
record stop
codegen playwright  # or:  codegen consoleena
output — recorded.spec.ts
import { test, expect } from '@playwright/test';

test('recorded flow', async ({ page }) => {
  await page.goto('https://your-app.test/');
  await page.locator('#email').fill('ada@example.com');
  await page.locator('text=Sign in').click();
  await expect(page.locator('.dashboard')).toBeVisible();
});

/AI agent

Prefix any command with ? (or switch the panel to ai mode). The agent snapshots the page, plans shell commands, observes the results, and iterates — up to config.agent.maxSteps rounds.

panel or console
# in the panel (prefix ?)
? log in as test@x.com / hunter2 and confirm the dashboard loads
? what globals look like they might be auth tokens

# JS API
await cona.ai('fill the signup form and assert success')

# debug helpers
fix             # diagnose recent JS errors via the LLM
fix --apply     # diagnose and eval the proposed patch live
explain .card   # explain an element in plain English
i
The default endpoint is keyless pollinations.ai — convenient but rate-limited. For real work, point it at any OpenAI-compatible API:

cona.config.llm.endpoint = 'https://api.openai.com/v1/chat/completions'
cona.config.llm.apiKey = 'sk-…'

/GitHub PR from a prompt

Describe a change in plain English. Consoleena asks the LLM for complete file contents, shows a preview, then creates the branch, commits every file, and opens the pull request — through the GitHub REST API, entirely client-side.

terminal
# one-time setup (PAT needs repo scope, or fine-grained Contents + PR write)
gh login ghp_yourtoken
gh repo suryasticsai/Consoleena#main

# raise a PR from a description
pr "add a footer with copyright and a GitHub link"
pr "refactor the Cart class to use a Map internally" --yes
!
The agent will not run gh or pr on its own — it proposes them and waits for confirmation, unless you set CONFIG.agent.autoApprove = true.

/safety & limits

!
Dev / staging only. The shell executes arbitrary JS, mutates globals, and with a token pushes to GitHub. Never load it in production where real user data or credentials are present.
  • Tokens live in the browser — visible to page scripts. Use a short-lived, minimally-scoped PAT. Persisting to localStorage is off by default.
  • AI features send page context to your LLM endpoint. Be mindful on pages with sensitive data.
  • Screenshots need html2canvas for real PNGs; SVG fallback otherwise.
  • Cross-origin stylesheets and iframes are not readable — browser security applies.
